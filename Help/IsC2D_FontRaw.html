FontRawInit(ID, *Memory.Union=#PB_Default, Length=0, zw=1, zh=1, Color.l=#White, Width=8, NumChar.a=#NUM_CHR) = #True, #Null = Fehler. Initialisiert einen Demo-/Cracktro-typischen RawFont wie er zu Zeiten des Amiga verwendet wurde. *Memory = Daten des RawFont. Als Standard = #PB_Default wird TopazNew.raw/8 initialisiert, hierzu muss #IsC2D_Topaz aktiviert sein. Length = bestimmt die Byte-Größe von *Memory. zw/zh = Höhe-/Breite pro Pixel-Bit. Color = RGB Farbwert. Width = tatsächliche Breite des RawFont. Die RawFont-Daten liegen in Byte (8 Bit), Word (16 Bit), Long (32 Bit) oder Quad (64 Bit) je Buchstaben-Breite vor. Sollte bei Byte (8 Bit) nur eine 5 Bit Breite verwendet werden, kann über Width eine entsprechende Anpassung erfolgen. NumChar = Anzahl der Zeichen (0-255) derzeit sollten immer 64 Buchstaben (entsprechend dem Standard-String) verfügbar sein!
Amiga Utilities (WinUAE)Beschreibung
F2R, F2RLcase, F2RUase Font2Raw (F2R) v1.0 von Niels Keurentjes ist ein nützliches Tool um RawFonts auf den Amiga zu erstellen. Da jedoch nach jeder RawFont-Erstellung nicht nur das Programm neu gestartet werden muss auch entsprechende Zeichenkette (64 Zeichen) jeweils eingetippt werden müssen da die Standardfolge nicht mit C2D kompatibel ist, wurde F2R zur Nervenschonung so gepatcht, daß F2RLcase bei Neustart einen C2D kompatiblen String in Kleinbuchstaben- bzw. F2RUcase in Großbuchstaben direkt zur Verfügung stellt. Zur Zeit unterstützt C2D nur das Byte-Format da wohl ein Word/Long Bug in F2R vorhanden ist?
